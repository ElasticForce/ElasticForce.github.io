146. LRU缓存机制

运用你所掌握的数据结构，设计和实现一个 LRU (最近最少使用) 缓存机制。它应该支持以下操作： 获取数据 get 和 写入数据 put 。

获取数据 get(key) - 如果关键字 (key) 存在于缓存中，则获取关键字的值（总是正数），否则返回 -1。
写入数据 put(key, value) - 如果关键字已经存在，则变更其数据值；如果关键字不存在，则插入该组「关键字/值」。当缓存容量达到上限时，它应该在写入新数据之前删除最久未使用的数据值，从而为新的数据值留出空间。

进阶:

你是否可以在 O(1) 时间复杂度内完成这两种操作？

思路：

LRU最近最久未使用算法，是用于操作系统发生缺页中断时，需要淘汰内存中的一个页面，给新的页面腾出位置。

以前解决这道问题利用的是链表，可以实现节点的快速移动，但是链表的查询速度是O(n)。

所以更好的选择是链式哈希表，查询速度变成了O(1)，存储顺序按照插入顺序排列，所以每次插入一个新的数据都在链尾。

当获取新的数据时，就移除该数据，同时插入链尾。

460. LFU缓存

请你为 最不经常使用（LFU）缓存算法设计并实现数据结构。它应该支持以下操作：get 和 put。

  • get(key) - 如果键存在于缓存中，则获取键的值（总是正数），否则返回 -1。
  • put(key, value) - 如果键已存在，则变更其值；如果键不存在，请插入键值对。当缓存达到其容量时，则应该在插入新项之前，使最不经常使用的项无效。在此问题中，当存在平局（即两个或更多个键具有相同使用频率）时，应该去除最久未使用的键。

「项的使用次数」就是自插入该项以来对其调用 get 和 put 函数的次数之和。使用次数会在对应项被移除后置为 0 。

LFU比LRU更难实现，如果仅仅是要求频率，那么使用两个哈希表，一个存key_value，一个存key_freq即可。但是LFU同时要求，频率相同时删除最早的key，等同于还要实现LRU的效果。

所以使用额外的哈希表，freq_keys，通过频率记录keys，对于相同频率的key，存储在一个有序集合上，删除时优先删除链表集合头部的最早key。
